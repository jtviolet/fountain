Project Ideas
Having a hard time coming up with a project on your own? Here are some ideas created by your peers that will have a direct benefit to FireEye. If you find a project you like you can reach out to the sponsor to get more details.
Performance graph in the Endpoint Security console
Sponsor : Endpoint PM
Description : By using XAPerf (internal tool) we can take snapshots of the endpoint data. This could be used to build out charts and trends on endpoint data.
Benefits : Customer's IT group consistently blames Endpoint Security as the offending app, where the SOC has to prove the agent was not the cause of a performance hit at a specific point of time.
Generic Reports
Sponsor : Endpoint PM
Description : Endpoint Console is in need of charts to show more insightful data to the analysts. Types of alerts, agent versions deployed, scan reports, etc… are all generated by hand through CSV downloads.
Benefits : Customers ask for reporting capabilities, either by executive reports, weekly alert summaries, or even agent population reports.
Generic Audit Viewer
Sponsor : Endpoint PM
Description : Consulting creates a lot of new scripts to be used by the agent, but the Endpoint Console cannot dynamically display these results as the Audit Viewer is hardcoded per audit type. Having a way to generically populate new audit data could be used as a module for all Mandiant/Manage Defense scripts. These scripts are finding the latest ways threat actors are evading detection.
Benefits : Customers will immediately receive scripts as they are vetted by Consulting versus having to wait for several months to years before engineering finds available cycles to add it in.
Storytime to be FireEye skinned
Sponsor : Endpoint PM
Description : Consulting created a feature called Storytime and will be used for the FireEye platform in a year or so. It will make its first appearance in Endpoint Security. Having it modernized to the MUSE fabric and adding any FireEye decorators will help speed up its release. This is the "WOW" factor when showing off your technology to prospective clients.
Benefits : Major increase in sale opportunities for Small and Medium business that do not have an experienced analysts to decipher and analyze the data. This visualization makes it much easier to understand what has occurred and the next question will be "What do I do next"
Mobile Forensics
Sponsor : Endpoint PM
Description : Customers ask for mobile support. The first pathway to manage a mobile device through Endpoint Security is to make a forensics feature on Android or iOS. Sluethkit is used by Endpoint Security today and could be extended to the mobile device too.
Benefits : Addresses a gap in mobile visibility while doing an investigation. Customers ask for protection, but Google and Apple do this fairly well with their app markets. An MDM will handle side loaded apps, so a gap to cover here is forensics, which a customer will be willing to install with their MDM solution.
Container Protection
Sponsor : Endpoint PM
Description : Container detection is coming in 2H 2020, but customers do need a level of protection. By providing a way to contain or respond to a threat inside of a container, docker, kubernettes, etc… environment will help leap over our vendors who only offer detection capabilities.
Benefits : Addresses a gap in container support where FireEye is missing a feature to enable Consulting and customers on stopping further attacks. This will expand our footprint in the Linux space.
Push capability from the Endpoint Console to Helix and other SIEMs
Sponsor : Endpoint PM
Description : Endpoint Console stores events and alerts on a Kafka message bus. Today, Helix and SIEMs have to poll for data every 10 to 30 minutes. Customers want the data in real-time, so a push mechanism would be needed.
Benefits : Customers have a wide variety of locations to push this JSON data too, such as Helix, Splunk, Elastic, ArcSight, etc… Extra points of you can make a connector that parses the JSON data to the SIEM event schema of their choice.
Exploit Detection Validation by Verodin
Sponsor : Endpoint PM
Description : Having Verodin ran against the Endpoint Security agent so we can check if the agent is running properly. This could be done by a module and point to a Verodin server for the results.
Benefits : Doing this check will have immense benefits. You can do this for a POV (Proof of Value) as the prospect can see alert data generated by the agent. We can use this for Support, to make sure the agent is reporting alerts according to features enabled. Customers can validate the agent is running as expected for a weekly report. This also enables more Verodin sales because now Verodin could be extended as part of an Endpoint subscription. It can also get customers to switch from their legacy AV, if they consistently see Verodin bypassing their security where Endpoint Security is only enabled for the EDR side.
MITRE Map on Console
Sponsor : Endpoint PM
Description : Shows what detections have triggered in your environment, so you can map out the agent capabilities.
Benefits : FORCE, Support, Manage Defense and Intel get an abundance of requests on Endpoint detection capabilities, where it is very difficult to determine without research. Having a MITRE map on the console can help customers understand what current detections they have already triggered upon (red) and possibly have a content feed to see what detections we support (green). This limits the confusion and shows our strength in MITRE.
AV deployment Audit
Sponsor : Endpoint PM
Description : Running an audit to find all possible 3rd party endpoint security vendors that are deployed and which ones are loaded. You can get this on Windows with WSC as the primary AV vendor needs to register to run.
Benefits : Many companies have multiple AV's running, so having a 10,000 foot view on which AV's are working vs not, such as FireEye finding a ton of threats the AV missed. Helps enable an AV replacement for FireEye to take over. Also it will feed our telemetry so we can study which AV's are being bypassed for better protection opportunities and to consider automatic best practices for stronger security controls if we know that AV is installed.
Asset Management
Sponsor : Endpoint PM
Description : Using Helix to find which devices do not have Endpoint Security installed. You can find this data from the Endpoint Security Console via API.
Benefits : Customers consistently want a report on which environments do not have Endpoint Security deployed too.
URL enabled agent downloads
Sponsor : Endpoint PM
Description : Landing page for endpoint users to click on an agent for an immediate install on their device.
Benefits : Customers have to enable SCCM, GPO, JAMF, or other deployment capabilities on getting the initial Endpoint Security agent installed to a new device. Having a static URL for customers to click on that gets them to a hosted location on the HX Console would be a quick win. Bonus points if you can tie the asset management detection and this URL download together.
Vulnerability Assessment
Sponsor : Endpoint PM
Description : Taking our current audits for deployed software and extending it to look at OS packages and plugins (ie. browers plugins) so we can match CPE's against it.
Benefits : Our Vulnerability Services team creates CVE's and can map CPE to CVE's. If we can run an audit or even better, run a scan with a list of known CPE's, then we could extend new opportunities for our customers between Endpoint Security and the Vulnerability Assessment team.
